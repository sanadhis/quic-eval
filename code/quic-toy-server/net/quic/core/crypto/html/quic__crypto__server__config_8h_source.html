251  // Checks |client_hello| for gross errors and determines whether it can be
252  // shown to be fresh (i.e. not a replay). The result of the validation step
253  // must be interpreted by calling QuicCryptoServerConfig::ProcessClientHello
254  // from the done_cb.
255  //
256  // ValidateClientHello may invoke the done_cb before unrolling the
257  // stack if it is able to assess the validity of the client_nonce
258  // without asynchronous operations.
259  //
260  // client_hello: the incoming client hello message.
261  // client_ip: the IP address of the client, which is used to generate and
262  // validate source-address tokens.
263  // server_address: the IP address and port of the server. The IP address and
264  // port may be used for certificate selection.
265  // version: protocol version used for this connection.
266  // clock: used to validate client nonces and ephemeral keys.
267  // crypto_proof: in/out parameter to which will be written the crypto proof
268  // used in reply to a proof demand. The pointed-to-object must
269  // live until the callback is invoked.
270  // done_cb: single-use callback that accepts an opaque
271  // ValidatedClientHelloMsg token that holds information about
272  // the client hello. The callback will always be called exactly
273  // once, either under the current call stack, or after the
274  // completion of an asynchronous operation.
275  void ValidateClientHello(
276  const CryptoHandshakeMessage& client_hello,
277  const QuicIpAddress& client_ip,
278  const QuicSocketAddress& server_address,
279  QuicTransportVersion version,
280  const QuicClock* clock,

282  std::unique_ptr<ValidateClientHelloResultCallback> done_cb) const;

284  // ProcessClientHello processes |client_hello| and decides whether to accept
285  // or reject the connection. If the connection is to be accepted, |done_cb| is
286  // invoked with the contents of the ServerHello and QUIC_NO_ERROR. Otherwise
287  // |done_cb| is called with a REJ or SREJ message and QUIC_NO_ERROR.
288  //
289  // validate_chlo_result: Output from the asynchronous call to
290  // ValidateClientHello. Contains the client hello message and
291  // information about it.
292  // reject_only: Only generate rejections, not server hello messages.
293  // connection_id: the ConnectionId for the connection, which is used in key
294  // derivation.
295  // server_ip: the IP address of the server. The IP address may be used for
296  // certificate selection.
297  // client_address: the IP address and port of the client. The IP address is
298  // used to generate and validate source-address tokens.
299  // version: version of the QUIC protocol in use for this connection
300  // supported_versions: versions of the QUIC protocol that this server
301  // supports.
302  // clock: used to validate client nonces and ephemeral keys.
303  // rand: an entropy source
304  // compressed_certs_cache: the cache that caches a set of most recently used
305  // certs. Owned by QuicDispatcher.
306  // params: the state of the handshake. This may be updated with a server
307  // nonce when we send a rejection.
308  // crypto_proof: output structure containing the crypto proof used in reply to
309  // a proof demand.
310  // total_framing_overhead: the total per-packet overhead for a stream frame
311  // chlo_packet_size: the size, in bytes, of the CHLO packet
312  // done_cb: the callback invoked on completion
