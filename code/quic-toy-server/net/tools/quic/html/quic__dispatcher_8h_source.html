175  // Values to be returned by ValidityChecks() to indicate what should be done
176  // with a packet. Fates with greater values are considered to be higher
177  // priority, in that if one validity check indicates a lower-valued fate and
178  // another validity check indicates a higher-valued fate, the higher-valued
179  // fate should be obeyed.
180  enum QuicPacketFate {
181  // Process the packet normally, which is usually to establish a connection.
182  kFateProcess,
183  // Put the connection ID into time-wait state and send a public reset.
184  kFateTimeWait,
185  // Buffer the packet.
186  kFateBuffer,
187  // Drop the packet (ignore and give no response).
188  kFateDrop,
189  };
